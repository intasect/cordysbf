ISV

Description

Creates Cordys BCP ISV package based on the given isv content xml file

Contents are used to specify the content that should be shipped with isvp.

Parameters

Attribute Description Required
owner Name of the owner of the ISV Package. Yes
productname Name of the product that will be contained in the ISV Package. Yes
version Version of the product being created. Yes
type The type of the ISV Package. The value could be 'standard' or 'componentized'. Yes
destdir The destination directory where the ISV package should be placed after creation. Yes
eulafile The File class which will provide the path of the End User License Argreement(EULA). No
imagefile The file which will provide the path of the Side bar image shown while uploading the ISV package. No
filename The name of file which is created by the ISV. The name doesn't contain extension of the file. No
isvDN Attribute which stores the distiguished name(dn) of the isv. No

Examples

 <isv
 	owner="${isv.owner}"
 	productname="${isv.owner}"
 	version="${isv.version}"
 	type="${isv.type}"
 	name="${isv.name}"
 	filename="${isv.filename}"
 	eulafile="{src.isv}/content/eula.html"
 	imagefile="{src.isv}/content/sidebar.bmp"
 	destdir="${dist.dir}">
 		<content type="roles" contentfile="${src.isv}/content/roles.xml" />
 		<content type="filesystem" contentfile="${src.isv}/content/filesystem.xml" />
 		<content type="methodsets" contentfile="${src.isv}/content/methodsets.xml" />
 		<content type="menus" contentfile="${src.isv}/content/menus.xml" />
 		<content type="systemenvironment" contentfile="${src.isv}/content/sysenv.xml" />
 		<content type="prompts" contentfile="${src.isv}/content/prompts.xml" />
 		<content type="xmlstore" contentfile="${src.isv}/content/xmlstore.xml" />
 		<content type="applicationconnectors" contentfile="${src.isv}/content/connectors.xml" />
 		<content type="styles" contentfile="${src.isv}/content/styles.xml" />
 		<content type="dbschema" contentfile="${src.isv}/content/dbschema.xml" />
 		<content type="virtualdirectories" contentfile="${src.isv}/content/virtualdir.xml" />
 		<content type="custom" handler="com.cordys.cpc.ant.BusinessObjectLoader" contentfile="${src.isv}/content/cobocdata.xml" />
 </isv>
