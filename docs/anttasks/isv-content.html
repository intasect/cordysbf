Content

Description

Content refers to a specific type of content that should be shipped with isvp.

Some of the content types have default handlers. If you want to write your own handler, you can override the default handler by using handler property. For new types (we call it as non-standard), you will have to write your own handler. Except for types 'custom' and 'static', you can have only one occurance of type nested within an isv task. Type 'custom' is a non-standard type. It doesn't have a default handler.

Parameters

Attribute Description Required
type Type of the content that you need to ship with isvp. Yes
handler Java class that implement content handler for specific type. For standard types this is optional. Required for non standard types.
Note: For standard types, you can use this to override the default handler. (Not yet implemented) 		Yes/No
contentfile File where specification of the content is given. This file should contain valid xml. Yes

Possible value for attribute type

Value Description
filesystem To specify the file system contents that needs to be shipped with ISV.
systemenvironment To specify the system environment variables that needs to be shipped with ISV.
roles To specify the roles that needs to be shipped with ISV.
methodsets To specify the methodsets that needs to be shipped with ISV.
menus To specify the menus that needs to be shipped with ISV.
applicationconnectors To specify the application connector definitions that needs to be shipped with ISV.
styles To specify the styles that needs to be shipped with ISV.
xmlstore To specify the xml store contents that needs to be shipped with ISV.
virtualdirectories To specify the virtual directories that needs to be created at the BCP server.
prompts To specify the prompts that will be taken during the installation of ISV.
static To specify some static xml content that needs to be inserted as such in the content definition of ISV.
custom To implement your own handler for some custom content. You can send the patch/file to BCP build framework so that it will be made a standard type in the future releases.
